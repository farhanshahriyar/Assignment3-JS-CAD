Assignment 3 Reflection

Challenges Encountered

Throughout this assignment, I encountered several challenges that tested my understanding of JavaScript fundamentals. One of the earliest difficulties arose in Part One, where I had to use the document.write() method to dynamically inject student information into the page while applying inline CSS styling entirely from within the JavaScript file, since modifying the HTML file was not permitted. Getting the positioning right using position: fixed and aligning the content at the bottom-left corner of the page required some trial and error. Additionally, implementing the character encoding function using String.charCodeAt() and String.fromCharCode() was tricky at first, as I needed to carefully iterate through each character of multiple strings and shift their character codes by exactly one position without breaking the output format.

Troubleshooting and Debugging

My approach to debugging relied heavily on the Mocha and Chai testing framework that was integrated into the assignment. By opening assignment3.html in the browser and keeping the developer console visible, I could immediately see which test cases passed and which ones failed. When a test failed, I would carefully re-read the expected output and compare it against what my function was returning. For example, in Part Three, I initially had trouble with the "Magic" prefix logic because I was checking divisibility by three before constructing the full concatenated string. Tracing through the loop iterations manually on paper helped me realize I needed to accumulate the sum separately while building the string, and only prepend "Magic" after the loop was complete. I also used console.log() statements to print intermediate variable values, which proved invaluable for catching off-by-one errors in my loops.

The Relationship Between HTML, CSS, and JavaScript

This assignment significantly deepened my understanding of how HTML, CSS, and JavaScript work together in web development. In Part One, I experienced firsthand how JavaScript can serve as a dynamic engine that generates HTML content on the fly. By embedding HTML tags such as <b>, <br>, and <div> within JavaScript strings passed to document.write(), I was able to create structured, styled content without ever touching the HTML file itself. The inline CSS within those generated elements allowed me to control layout and positioning purely from within JavaScript. This demonstrated to me that JavaScript acts as a powerful bridge between structure and presentation—it can dynamically produce HTML markup and attach CSS styles at the same time. Understanding this relationship is essential because modern web applications depend on this interplay to create interactive, data-driven user interfaces.

The Most Challenging Aspect

The most challenging part of the assignment was undoubtedly Part Five, which required rearranging an array by separating odd and even numbers, maintaining their original relative order, and then appending the sums of odd and even values at the end of the new array. Keeping track of multiple temporary arrays and running totals simultaneously demanded careful organization. I overcame this challenge by breaking the problem down into clear, sequential steps. First, I focused on iterating through the input array once using the modulo operator to classify each number as odd or even, pushing them into separate arrays while accumulating their respective sums. Next, I built the result array by combining the odd numbers first, followed by the even numbers, using a simple loop-based approach. Finally, I used the push() method to append the odd sum and even sum to the end of the result array. Part Four also presented interesting edge cases, such as handling invalid array indices and distinguishing between a value of zero, even values, and odd values at a given index. Taking a methodical, step-by-step approach to each part prevented me from feeling overwhelmed and allowed me to build a solid, working solution for every function in the assignment.
