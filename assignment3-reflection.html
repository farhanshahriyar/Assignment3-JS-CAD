Assignment 3 Reflection

Challenges Encountered

While completing this assignment, a few unique challenges surfaced that required careful attention to detail. One of the primary obstacles was ensuring that the string formatting matched the exact specifications of the automated unit tests. For instance, in Part Three, determining exactly when to prepend the string "Magic" versus using all capital letters required me to closely read both the assignment hints and the automated testing logs. Additionally, handling the text encoding in Part One presented an interesting hurdle. Utilizing the charCodeAt() and String.fromCharCode() methods to perfectly offset characters by a numerical value, while still preserving the complex structural formatting of the original plain-text document, demanded a nuanced understanding of JavaScript string manipulation.

Troubleshooting and Debugging

My strategy for troubleshooting heavily relied on the provided Mocha automated unit testing framework. By keeping the browser console open and frequently running the local testing environment, I could instantly receive logical feedback on whether my conditional statements and loops were behaving as anticipated. Whenever a testing bullet failed—such as an array index out of bounds error or a mismatched return string—I carefully traced my variables backward. Tracking loop iterations and isolating distinct logical branches (such as prioritizing the evaluation of a 0 value versus checking for even numbers in Part Four) helped me to squash bugs systematically rather than relying on guesswork.

The Relationship Between HTML, CSS, and JavaScript

This assignment profoundly deepened my understanding of how HTML, CSS, and JavaScript collaborate to form a cohesive web interface. Specifically, in Part One, I learned how JavaScript can act dynamically as the engine that not only calculates background logic but actively paints the DOM. By passing string literals into document.write(), I was able to inject raw HTML tags like line breaks and bold tags directly onto the screen. Furthermore, I learned to embed inline CSS directly within the JavaScript string to fix the position of the student information block securely to the bottom right corner of the window. This demonstrated that JavaScript behaves as the foundational bridge that strings style and structure together dynamically.

The Most Challenging Aspect

By far, the most challenging aspect of the assignment was Part Five, which required traversing an array, separating its values by mathematical parity, and dynamically rebuilding a newly structured array with calculated sums pushed to its end. Maintaining multiple temporary data structures simultaneously without dropping elements required immense focus. I overcame this by methodically breaking the complex problem down into sequential steps. First, I focused solely on iterating through the list once and cleanly dividing the numbers using the modulo operator while tracking the separated sums. Once that foundation was stable, I leveraged the built-in concat() method to merge the odd and even arrays cleanly. Finally, I utilized the push() method to seamlessly append the calculated sums to the dynamically lengthened array. Tackling the problem in isolated clumps prevented me from feeling overwhelmed and ultimately cemented a robust algorithm.
